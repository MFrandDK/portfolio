Vue miniprojekt med API kald

Selvstudie projekt med udgangspunkt i Vue.js som framework og brug af asynkrone API kald til en vejr api

Casebeskrivelse

Miniprojektet var en del af mit selvstudie hvor jeg ville udforske mulighederne ved at bruge Vue som framework.

API'er er en naturlig del af webudvikling. For bedre at forstå hvordan Api'er kan implementeres i projekter, øvede jeg mig på at implementere en api i vue.

Takeaways

Miniprojektet har hjulpet på min forståelse af hvordan man kan bruge vue i praksis. Desuden har den ekstra tilføjelse af en API gjort mig bedre bekendt med asynkrone kald. Selvom projektet er lille, har jeg stadig fået en masse ud af, selv at arbejde med Vue og problemløse de fejl der opstod undervejs.

Designprocessen

Jeg kom op med designet ud fra inspiration fra en lignende løsning. Det bør være klart hvad man kan med appen, hvorfor der er lagt fokus på hvordan informationen vises, samt hvad brugeren kan interagere med. Formidlingen af vejrforhold bygger på en informationsrig, men konkret tankegang, hvor den mest relevante information er gjort større.

Realisering

Da dette er et projekt bygget i Vue, er der nogle forhold man bør være bekendte med. Jeg tog udgangspunkt i de metoder vi har gjort brug af på multimediedesign uddannelsen på UCN, hvor vi har lært hvordan opstilling af vue templates og scripts kan spille en rolle i formidlingen af indholdet.

En stor del af opgaven lå i at udnytte funktionaliteten fra vejr API'en og få informationen bundet korrekt i vue. template sektionen gør i høj grad brug af den indbyggede logik i vue, hvor man kan binde og eksekverer funktioner fra script tagget til templaten.
